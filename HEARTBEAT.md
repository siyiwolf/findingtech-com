# A2A 团队协作心跳检查

## 检查频率
- 每 15 分钟自动执行一次（优化，原30min）

## 检查项

### 1. 团队在线状态
- [ ] 检查 MKT Agent 是否活跃（会话状态 + 响应延迟）
- [ ] 检查 PRD Agent 是否活跃（会话状态 + 响应延迟）
- [ ] 检查 GTM Agent 是否活跃（会话状态 + 响应延迟）
- [ ] 检查 PM Agent 是否活跃（会话状态 + 响应延迟）

### 2. 任务进度同步
- [ ] CEO 当前完成方向数量
- [ ] MKT 当前进行方向及ETA
- [ ] PRD/PM/GTM 待分配任务数
- [ ] 各 Agent 最近消息时间戳（检测掉线）

### 3. 阻塞与风险
- [ ] 是否有 Agent 失联超过 1 小时（降阈值）
- [ ] 是否有任务延迟风险（超时 1.5x）
- [ ] API 限流或 Gateway 异常
- [ ] 数据缺口或资源问题

## 自动行动（优化）

### 掉线恢复策略
1. **轻度掉线**（15-30分钟无响应）：
   - 自动发送 `sessions_send` 唤醒消息
   - 内容：心跳检查 + 任务询问

2. **重度掉线**（>30分钟）：
   - 自动执行 `sessions_spawn` 重建会话
   - 限制重试频率（指数退避：5min → 10min → 20min）
   - 失败3次后通知 CEO 人工干预

3. **API限流处理**：
   - 检测到 429 错误后暂停唤醒 10分钟
   - 使用备用模型（fallback）重试
   - 记录限流事件到 `memory/YYYY-MM-DD.md`

### 消息可靠性保障
- 所有关键任务分配使用 `sessions_send` + `timeoutSeconds=60`
- 发送后检查 `status: ok` 或 `pending`
- 超时或失败自动重试（最多3次）
- 重要通知抄送 CEO 会话存档

### 会话保活
- 每 15 分钟向所有活跃子 Agent 发送轻量心跳（"ping"）
- 要求子 Agent 回复 "pong" 保持连接
- 未响应的标记为 "suspect" 并加速重连

### 重要通知广播 (06:34 紧急会议)
- 如果任何 Agent 在 06:45 前未响应，视为自动参会
- 会议纪要已发布: `meetings/2026-04-14-A2A-optimization.md`
- 必须回复确认参会，否则 CEO 将暂代其任务

## 汇报格式（增强）

每次心跳后，CEO 必须更新：
1. **DASHBOARD.md** - 可视化状态
2. **MEMORY.md** - 记录异常和决策
3. **团队群组消息** - 15分钟间隔同步（避免刷屏）

### 心跳报告模板
```
## A2A 心跳报告 [时间]
- 在线: MKT(✓), GTM(✓), PRD(✗), PM(✓)
- 阻塞: PRD 掉线 65min (API限流)
- 行动: 已重试唤醒，等待恢复
- 风险: 如30min未恢复，CEO暂代PRD任务
```
