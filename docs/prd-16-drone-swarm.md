## 1. 产品概述

### 1.1 产品定义

无人机集群系统（Drone Swarm System）是一套基于协同控制算法的多无人机作业解决方案，用于户外场景的自动化任务执行，如营地巡检、景区表演、工业检测、应急救援等。系统由一个集群控制站（SwarmHub）和 5-50 架次无人机组成，实现多机协同、路径规划、自主避障、任务编排的一体化管理。

### 1.2 目标市场

- **露营/营地运营商**：营地安全巡检、夜间灯光表演、游客引导
- **景区管理方**：观光导览、空中表演、安全监控、人流统计
- **工业检测公司**：电力线路、管道、桥梁、大型基础设施巡检
- **农业企业**：植保、农田测绘、作物监测
- **应急救援机构**：搜索救援、灾害评估、物资投送

### 1.3 核心价值主张

| 价值点 | 说明 | 量化指标 |
|--------|------|----------|
| 效率提升 | 多机协同，单次覆盖大范围 | 作业效率相比单机提升5-10倍 |
| 成本降低 | 共享控制平台，减少人工 | 人力成本降低70%，单机成本降低30% |
| 安全性 | 全自动飞行，减少人员风险 | 零伤亡事故（对比人工高空作业） |
| 可扩展性 | 模块化集群，按需增减 | 支持5-50架弹性扩容 |
| 智能化 | AI任务规划与自主避障 | 路径优化提升20%飞行效率 |

### 1.4 定价

- **基础版**：¥800,000（含10架无人机 + SwarmHub + 基础软件）
- **企业版**：¥2,000,000（含25架 + 高级算法 + 5G通信 + 2年维护）
- **定制版**：¥5,000,000+（按需定制规模、特殊负载、私有化部署）
- **服务套餐**：¥200,000/年（7x24支持、软件升级、定期演练）

## 3. 功能规格

### 3.1 SwarmHub（集群控制中心）

**功能**：
- 集群状态监控（GPS、电量、任务进度）
- 任务规划与路径分配（AI优化）
- 多机通信协调（自组网 + 5G/卫星冗余）
- 应急处理（一键返航、悬停、避障）
- 数据记录与分析（飞行日志、视频回传）

**性能**：
- 最大支持同时控制：50架
- 控制刷新率：10Hz
- 通信距离：5km（自组网）/ 无限制（5G）
- 延迟：< 200ms

---

### 3.2 Drone-Standard（标准集群无人机）

**硬件**：
- 尺寸：450mm轴距
- 重量：4kg（含电池）
- 最大载重：1.5kg
- 续航：45分钟（标准负载）
- 防护等级：IP54（防尘防水）
- 抗风等级：6级风（10m/s）
- 定位：GPS + GLONASS + 视觉定位（无GPS可用）

**功能**：
- 自动任务执行（航线飞行、定点监控、区域扫描）
- 集群协同编队（编队飞行、相对定位）
- 实时避障（前/下/上视觉 + 超声波）
- 应急处理（低电量返航、断联悬停）
- 负载接口（通用快拆，支持相机、喊话、抛投等）

---

### 3.3 集群算法

**核心能力**：
- **协同控制**：分布式群体智能，无中心单点故障
- **路径规划**：多目标优化，避免交叉和碰撞
- **通信协议**：基于LoRa的自组网协议（2.4GHz），Mesh网络
- **自主避障**：基于视觉+超声波的多层检测
- **故障隔离**：单机故障自动剔除，重新分配任务

**性能指标**：
- 集群规模：5-50架
- 避障响应：< 100ms
- 相对定位精度：< 0.5m（集群内）
- 消息吞吐：1000 msg/sec (10hz for 100 drones)

---

### 3.4 通信协议

- **无人机间**：LoRa自组网（2.4GHz），TDD，加密传输
- **无人机 ↔ SwarmHub**：5G + LoRa冗余双链路
- **SwarmHub ↔ 云端**：5G/4G + gRPC over TLS
- **卫星备份**（可选）：铱星短报文用于应急状态上报
- **频率合规**：符合中国无线电管理规定

---

### 3.5 安全机制

- **防碰撞**：集群内相对定位 + 动态间距保持（最小10m）
- **电子围栏**：预设禁飞区（机场、政府机关），自动避让
- **应急返航**：低电量（<20%）、断联（>10秒）、异常震动
- **数据加密**：AES-256（控制指令、视频流）
- **鉴权**：每个无人机唯一证书，SwarmHub双因素认证
- **日志审计**：所有飞行记录不可篡改，保留2年

## 4. 技术架构

### 4.1 软件栈

```
┌──────────────────────────────────────────────┐
│           Cloud / Command Center             │
│   Web Console (Vue3) + API (Go)             │
│   数据分析、任务历史、设备管理              │
├──────────────────────────────────────────────┤
│           SwarmHub (Jetson Orin)            │
│   ROS 2 Humble + DDS (FastRTPS)            │
│   集群调度器 + 任务分配 + 监控              │
├──────────────────────────────────────────────┤
│           Comm Layer (5G / LoRa)            │
│   双链路通信，自动切换                      │
├──────────────────────────────────────────────┤
│           Drone Firmware (Pixhawk)         │
│   PX4 Autopilot + ROS 2 Bridge             │
│   局部路径规划 + 避障 + 状态反馈           │
└──────────────────────────────────────────────┘
```

---

### 4.2 系统接口

- **SwarmHub API**：
  - `POST /api/v1/tasks` 创建集群任务
  - `GET /api/v1/drones` 状态查询
  - `POST /api/v1/emergency` 应急控制

- **无人机遥测**：
  - 10Hz 位置/姿态/电量
  - 1Hz 相机视频（H.264）
  - 事件上报（障碍物、异常）

## 5. 用户旅程

### 5.1 购买与部署

```
1. 需求沟通（销售）
2. 现场勘查（技术团队）
3. 方案设计与报价
4. 合同签署（4-8周交付）
5. 现场安装（SwarmHub + 充电站 + 网络）
6. 培训（操作员3天，管理员1天）
```

**关键里程碑**：
- T+1周：硬件到场
- T+2周：系统联调
- T+4周：首次集群飞行
- T+6周：客户验收

---

### 5.2 日常使用

**任务类型**：
1. **巡检**：预规划航线，自动飞行，视频回传，违章识别
2. **表演**：编队演示，灯光/声音同步，音乐配合
3. **测绘**：区域扫描，正射影像，3D建模
4. **物流**：点对点运输，自动装卸

**操作流程**：
```
① 任务准备（导入地图、标记路径、设置参数）
② 集群唤醒（自动校准、自检、GPS锁定）
③ 一键起飞（自动执行，监控大屏）
④ 实时监控（异常弹窗、手动干预）
⑤ 自动返航（任务完成或手动终止）
⑥ 数据导出（视频、日志、报告）
```

---

### 5.3 维护与保养

**日常**：
- 电池充电与健康检查（自动）
- 螺旋桨磨损检查（每周）
- 固件OTA升级（季度）

**定期**：
- 每3个月：IMU校准、电机保养
- 每6个月：全面检测 + 性能测试
- 每年：关键部件更换（电池、电机）

**支持**：
- 7x24 远程诊断
- 2小时响应（ onsite）
- 备件库（常用易损件）
