## 1. 产品概述

### 1.1 背景
在自然灾害、户外探险、城市突发事件等场景，受困人员常面临通信中断、定位失效、医疗资源匮乏等问题。传统应急装备功能单一、携带不便，或依赖公共基础设施。应急救援套装旨在整合急救、通信、定位、供电四大核心功能，打造一款便携、自给自足、易于操作的智能装备，满足个人、家庭、救援团队在极端环境下的生存和救援需求。

### 1.2 目标
- 为用户提供**一体化**应急解决方案，覆盖生存、通信和救援核心需求。
- 确保在**无公共基础设施**（电力、通信网络）环境下正常工作。
- 提高**便携性**和**易用性**，确保非专业人员也能快速上手。
- 通过**模块化设计**，支持用户根据场景需求组合功能模块。

### 1.3 核心价值
| 价值主张 | 描述 |
|----------|------|
| 自给自足 | 内置供电模块和通信模块，无需依赖外部电力或网络 |
| 多场景适用 | 户外探险、自然灾害、城市突发事件 |
| 易于操作 | 直观操作界面和语音引导，紧急情况下快速使用 |
| 模块化设计 | 可按需选择或升级模块（电池、通信频段） |
| 高兼容性 | 支持与常见应急设备（卫星电话、GPS）连接 |

### 1.4 目标用户
| 用户群体 | 描述 | 场景 |
|----------|------|------|
| 户外爱好者 | 徒步、登山、露营，偏远地区安全保障 | 无信号区域求救 |
| 救援队伍 | 专业救援人员现场指挥与救援 | 灾害现场通信 |
| 家庭用户 | 应对自然灾害（地震、台风）或突发事件 | 家庭应急包 |
| 企业/机构 | 员工/学生紧急疏散与救援 | 批量采购与培训 |

### 1.5 使用场景
- **自然灾害**：地震、洪水、台风导致通信/电力中断，装备提供急救、定位、通信。
- **户外探险**：山区、沙漠等无信号区域，装备提供定位、SOS、电源。
- **城市突发事件**：停电、基站故障，装备作为临时通信和供电设备。
- **紧急疏散**：火灾、恐怖袭击，装备帮助快速定位和获取救援指引。

## 2. 用户故事与用例

### 2.1 用户故事
| 角色 | 需求 | 价值 |
|------|------|------|
| 户外爱好者 | 在登山受伤时，向救援中心发送位置和求救信息 | 提高获救概率 |
| 家庭用户 | 地震后被困，使用语音助手指导进行自救 | 减少伤亡 |
| 救援队伍 | 在无信号区域，通过卫星通信与指挥中心保持联系 | 提升救援效率 |
| 企业安全官 | 批量采购并为员工提供应急培训 | 履行安全责任 |

### 2.2 用例分析
1. **一键SOS**：用户按下SOS按钮 → 装备自动发送位置（北斗/GPS）和身份信息至救援中心（卫星短信）。
2. **语音急救指导**：用户触发“急救帮助” → AI语音播放CPR、止血等步骤（离线语音库）。
3. **多设备协同**：多个装备组成Ad-hoc网络，共享位置，形成临时通信网。
4. **电源共享**：装备可为手机、对讲机等设备充电（USB-C输出，≥10,000mAh）。

## 3. 功能规格

### 3.1 急救模块
- **内置物料**：止血带、敷料、消毒片、剪刀、急救毯、CPR面罩。
- **智能提醒**：物料使用记录 + 过期提醒（App同步）。
- **语音指导**：多语言急救步骤指导（离线）。
- **医疗认证**：符合国家急救包标准（GB/T 20557）。

### 3.2 通信模块
- **卫星通信**：支持北斗短报文（中国境内）+ GPS（全球定位）。
- **蜂窝备份**：4G Cat-M1/NB-IoT（有信号时自动切换）。
- **对讲功能**：模拟/DMR数字对讲（1-5km，无网络）。
- **SOS广播**：一键发送至当地救援中心（112国际紧急号码）。

### 3.3 定位模块
- **多模定位**：北斗/GPS/GLONASS三模，精度 ≤5米。
- **航迹记录**：存储最后72小时轨迹（本地存储）。
- **地理围栏**：进入危险区域自动告警（预设边界）。
- **室内定位**：可选配UWB模块（室内搜救）。

### 3.4 供电模块
- **电池**：可充电锂离子电池 12,000mAh（3.7V），支持-20℃~60℃。
- **太阳能**：外接柔性太阳能板（20W），光照3小时充50%。
- **手动发电**：手摇发电机（应急，5分钟通话）。
- **输出接口**：USB-A（18W PD）、USB-C（20W PD）、DC 12V。
- **续航**：待机 ≥30天；连续使用（通信+定位） ≥12小时。

### 3.5 硬件规格
- **尺寸**：180mm × 120mm × 60mm
- **重量**：≤ 800g（含电池和急救包）
- **防护**：IP67防水、 MIL-STD-810H 耐冲击
- **显示屏**：1.3" 黑白OLED（阳光下可视）
- **按键**：SOS（红色，保护盖）、电源、音量、PTT对讲
- **接口**：USB-C（充电/数据）、天线接口（SMA）
- **环境**：工作温度 -20℃~60℃，存储 -30℃~70℃

### 3.6 软件功能
- **本地UI**：图形化菜单，支持触摸+按键。
- **语音助手**：离线语音识别（急救询问、设备设置）。
- **蓝牙**：连接手机App（配置、轨迹同步）。
- **固件升级**：OTA（通过蓝牙/USB）。
- **安全**：加密通信（卫星短信AES-256），位置可设隐私模式。

## 4. 技术架构

### 4.1 系统架构
```
+-------------------+
|  用户界面层        | ← 按键 + OLED + 语音
+-------------------+
|  业务逻辑层        | ← 急救、通信、定位、供电管理
+-------------------+
|  硬件抽象层        | ← 卫星模组、GPS、电池、传感器
+-------------------+
|  支撑服务          | ← OTA、日志、安全
+-------------------+
```

### 4.2 核心组件
| 组件 | 型号/规格 | 供应商 | 备注 |
|------|-----------|--------|------|
| MCU | STM32H743 (Cortex-M7) | ST | 主控 |
| 卫星模组 | 北斗RDSS短报文模块 | 中国卫通 | 合规 |
| GPS | u-blox NEO-M9N | u-blox | 多模 |
| 电池 | 3.7V 12000mAh 锂离子 | ATL | 可充电 |
| 显示屏 | 1.3" OLED (128×64) | 国产 | 阳光下可视 |
| 对讲模块 | DMR数字对讲 | 海能达 | 1-5km |
| 4G模组 | Quectel BC35-G (NB-IoT) | 移远 | 备份通信 |
| 太阳能控制 | MPPT充电控制器 | 自研 | 20W输入 |

### 4.3 数据流
- **SOS流程**：用户按SOS → MCU收集位置 → 加密 → 卫星发送 → 救援中心。
- **航迹**：GPS每5秒记录 → 存储Flash → 蓝牙同步App。
- **充电**：太阳能/USB → MPPT → 电池管理 → 电池。

### 4.4 安全与合规
- 通信加密：AES-256
- 数据存储：本地加密
- 医疗物料：符合GB标准
- 无线电：取得型号核准（SRRC）

## 5. MVP 范围

### 5.1 MVP 功能列表
- 基础定位（北斗/GPS，精度≤10米）
- 卫星短信SOS（发送预设信息 + 位置）
- 急救包（13项基础物料）
- 语音急救指导（离线，10种场景）
- 电源输出（USB-C 18W，可为手机充电）
- 蓝牙配置App（iOS/Android）

### 5.2 排除项
- 视频通信（Pro版本）
- 多设备协同Mesh网络（后续版本）
- 4G蜂窝通信（成本考量，保留接口）
- 卫星双向通话（硬件成本过高）

### 5.3 验证指标
- SOS发送成功率 ≥ 99%
- 定位精度 ≤ 10米（开阔地）
- 电池续航 ≥ 12小时（连续使用）
- 操作学习时间 ≤ 3分钟（用户测试）
- 防水等级 IP67（30分钟 1米水深）
