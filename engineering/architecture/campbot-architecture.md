# CampBot 智能营地系统架构设计

> 多机器人协同，自动化营地搭建与管理

---

## 🎯 系统目标

- **全自动搭建**：从开箱到完成 ≤ 20分钟
- **多机协同**：6个机器人无缝协作，无冲突
- **环境适应**：户外复杂地形，-10°C~50°C，防雨防尘
- **低功耗长续航**：单次充电工作 ≥ 48小时
- **易用性**：App一键启动，无需专业培训

## 🔄 核心工作流程

### 1. 帐篷搭建流程（AssemblyBot + CarrierBot）

```
用户App选择帐篷类型 → 任务分解 → 机器人唤醒 → 协作完成

步骤：
1. CarrierBot #1 运送帐篷主体到指定位置
2. CarrierBot #2 运送地钉、锤子等配件
3. AssemblyBot 展开帐篷框架，使用视觉定位
4. AssemblyBot 打地钉（扭矩控制，防止损坏）
5. ClimateBot 监测环境，必要时启动除湿
6. 所有机器人报告完成状态
```

**并发策略**：
- CarrierBot #1 和 #2 同时搬运不同配件，路径规划避免碰撞
- AssemblyBot 等待配件到达后开始，实时进度反馈

### 2. 安全巡逻流程（GuardBot）

```
定时任务：每30分钟巡逻一次
- 路径：预设关键点（帐篷周界、厨房区、储藏区）
- 传感器：CO、烟雾、异常声音检测
- 异常处理：发现异常 → 通知ControlHub → App告警用户
```

### 3. 能源管理流程（PowerBot + ClimateBot）

```
太阳能板部署：
- 根据太阳角度自动调整倾角（优化发电）
- 电池电量监测，低电量时优先供电保障

温控流程：
- ClimateBot 监测帐篷内温湿度
- 超过阈值 → 启动加热/制冷片
- 结合太阳能发电情况调节功率
```

## 🧠 智能调度算法

### 问题
多机器人任务分配与路径规划，避免死锁和拥堵。

### 方案
- **任务分解**：Mission = {Task1, Task2, ...}
- **拍卖算法（ Auction-based）**：
  - 每个机器人根据能力（电量、位置、负载） bidding
  - 最高分获得任务，考虑整体最优
- **冲突检测**：实时交换位置信息，预留缓冲区
- **动态重调度**：机器人故障时，任务重新分配

### 实现
- 中央控制器（ControlHub）运行调度引擎（Python + OR-Tools）
- 机器人状态订阅（ROS2 topic）
- 任务指令发布（Action/Goal）

## 📡 通信协议

### 机器人 ↔ ControlHub
- **ROS 2 DDS**（最佳性能）
  - 主题（Topics）：状态发布（`/robot/state`）、传感器数据
  - 动作（Actions）：任务执行（`/mission/start`）
- **MQTT over TLS**（备用，跨网络）
  - 机器人状态 → Cloud → ControlHub（远程监控）

### ControlHub ↔ Cloud
- **gRPC**：设备管理、日志上传、OTA
- **HTTP/2**：文件传输（地图、模型）

### App ↔ Cloud
- **REST API**：任务创建、状态查询
- **WebSocket**：实时进度推送

## 🔐 安全机制

1. **设备认证**：每个机器人唯一证书（X.509），TLS双向认证
2. **指令签名**：关键指令（如打地钉）需用户二次确认
3. **急停机制**：硬件急停按钮 + 软件急停指令（广播）
4. **地理围栏**：机器人活动范围限制（防止跑偏）
5. **日志审计**：所有操作记录，不可篡改

- 搭建成功率 ≥ 95%
- 平均搭建时间 ≤ 20分钟
- 协同碰撞次数 = 0
