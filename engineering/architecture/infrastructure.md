## 🖥️ Cloud 设计（SwarmCloud）

### 容器编排
- **Kubernetes**：生产环境使用（EKS / ACK / 自建 K8s）
- **轻量级**：开发/小客户使用 Docker Compose 或 Nomad

### 微服务拆分

| 服务名 | 职责 | 技术栈 | 依赖 |
|--------|------|--------|------|
| `scheduler-service` | AI 任务调度核心 | Python + FastAPI + PyTorch | Redis, Kafka |
| `device-service` | 设备生命周期管理 | Go + Gin | PostgreSQL |
| `auth-service` | 认证与授权 | Go + OAuth2 | PostgreSQL |
| `api-gateway` | 统一入口、限流、路由 | Kong / APISIX | - |
| `websocket-service` | 实时推送 | Node.js + Socket.IO | Redis |
| `analytics-service` | 数据分析、报表 | Python + Pandas | PostgreSQL, TimescaleDB |
| `ota-service` | 固件升级管理 | Go | MinIO |
| `billing-service` | 订阅与计费 | Java / Go | PostgreSQL |

## 🖥️ Edge 设计（SwarmEdge）

### 硬件规格（推荐）
- **工控机**：Intel i5/i7, 32GB RAM, 1TB SSD, 双网口
- **网络**：千兆交换机，UPS 备用电源
- **部署**：客户仓库弱电间，物理安全

### 软件栈
- **OS**：Ubuntu 22.04 LTS
- **容器运行时**：Docker + K3s（轻量 K8s） or Docker Compose
- **SwarmAgent**：Go 编写，独立进程，Systemd 管理

### 高可用
- 双机热备（Active-Standby）
- Keepalived 虚拟 IP
- 共享存储（NFS）或数据库复制

## 🗃️ 数据存储选型

| 数据类型 | 存储引擎 | 理由 |
|----------|----------|------|
| 设备元数据、用户信息 | PostgreSQL | 关系型，ACID，复杂查询 |
| 时序数据（传感器、位置） | TimescaleDB（基于 PG） | 高效时间序列压缩与分析 |
| 缓存、会话、队列 | Redis | 高性能内存存储 |
| 对象存储（OTA 包、日志） | MinIO（S3 兼容） | 大文件，成本低，可替换 AWS S3 |
|  ELK 日志  | Elasticsearch + Logstash + Kibana | 日志搜索分析 |

## 🔐 网络安全

### 网络分区
```
Internet
   ↓ (TLS)
DMZ（反向代理、API Gateway）
   ↓ （内部网络）
应用层（SwarmCloud 微服务）
   ↓ （专线/VPN）
数据层（PostgreSQL, Redis 等）
   ↓ （客户内网）
Edge Layer（SwarmEdge）
```

### 安全措施
- **WAF**：Web 应用防火墙，防 SQL 注入、XSS
- **DDoS**：云服务商高防 IP
- **VPC**：私有网络，安全组限制端口
- **审计**：所有 API 调用记录，7 年保留
- **密钥管理**：HashiCorp Vault / AWS KMS

| 连接方向 | 协议 | 端口 | 加密 |
|----------|------|------|------|
| App → API Gateway | HTTPS / WSS | 443 | TLS 1.3 |
| API Gateway → 微服务 | HTTP/2 | 内部 | mTLS |
| SwarmEdge ↔ SwarmCloud | gRPC + TLS | 50051 | TLS 1.3 |
| SwarmEdge ↔ 机器人 | 厂商协议 | - | 可选（设备支持） |
| 机器人 ↔ 机器人 | (none) | - | 不直接通信 |
| 卫星通信 | 北斗/铱星 | - | 端到端加密 |
