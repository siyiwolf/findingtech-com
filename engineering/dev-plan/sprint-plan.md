# 敏捷开发冲刺计划

---

## 🏃 开发模式

- **框架**：Scrum 2.0（远程协作优化）
- **冲刺周期**：2 周（10 个工作日）
- **每日站会**：15 分钟，异步 + 同步结合
- **评审会**：冲刺最后一天，演示成果
- **回顾会**：冲刺结束后，改进流程

---

## 👥 团队组成（12 人）

| 角色 | 人数 | 职责 |
|------|------|------|
| **Product Owner** | 1 | 需求优先级，验收功能 |
| **Scrum Master** | 1 | 流程协调，移除障碍 |
| **Tech Lead** | 1 | 架构设计，代码审查 |
| **后端工程师** | 3 | 微服务、数据库、算法 |
| **嵌入式工程师** | 2 | 固件、驱动、ROS2 |
| **前端工程师** | 2 | Web UI + Flutter App |
| **测试工程师** | 2 | 单元测试、集成测试 |

**总人力**：12 人 × 2 周 = 240 人天

---

## 📋 冲刺计划（前 4 个冲刺）

### Sprint 0（准备期）：基础设施搭建
**目标**：完成开发环境、CI/CD、基础框架
- [ ] 搭建 GitHub / GitLab 仓库结构
- [ ] 配置 CI/CD（GitHub Actions）
- [ ] 建立监控（Prometheus + Grafana）
- [ ] 设计数据库 Schema
- [ ] 搭建 K8s 开发集群（Minikube / Kind）
- [ ] 编写项目脚手架（后端 / 前端 / 固件模板）
- **验收**：团队可以正常拉取代码、构建、部署到开发环境

---

### Sprint 1：设备管理 MVP
**目标**：设备注册、状态上报、基础控制
- [ ] 实现 `device-service`：设备 CRUD + 状态管理
- [ ] 实现 `auth-service`：JWT 认证、RBAC
- [ ] 设计设备影子（Device Shadow）规范
- [ ] SwarmAgent 基础框架：连接 Cloud、上报状态
- [ ] 固件：STM32 基础驱动 + FreeRTOS 任务框架
- [ ] Web UI：设备列表页面（只读）
- **验收**：设备可以在云注册，状态实时显示在 Web UI

---

### Sprint 2：任务调度原型
**目标**：单机器人任务分配与执行
- [ ] 实现 `scheduler-service` 基础版本（规则调度）
- [ ] 定义 Task 数据模型与 API
- [ ] 固件：导航功能（Nav2） + 简单避障
- [ ] Web UI：任务创建与手动分配
- [ ] 实现 WebSocket 推送任务状态
- **验收**：可以创建搬运任务，机器人执行并反馈位置

---

### Sprint 3：CampBot 多机协同
**目标**：6 机器人协作完成搭建任务
- [ ] 设计协同调度算法（拍卖算法原型）
- [ ] 实现任务分解（Mission → Tasks）
- [ ] 固件：AssemblyBot 机械臂控制（URDF 建模）
- [ ] 固件：CarrierBot 载重搬运
- [ ] Web/App：CampBot 任务一键启动界面
- [ ] 仿真测试：Gazebo 环境模拟 6 机器人
- **验收**：1 个帐篷完整搭建（Assembly + 2×Carrier + Climate）

---

### Sprint 4： Finding Pro 端侧 AI
**目标**：Risk预警与轨迹预测
- [ ] 嵌入式 AI 框架：TensorFlow Lite Micro
- [ ] 训练风险预警模型（天气、地形）
- [ ] 固件：传感器融合（GPS + IMU + 气压计）
- [ ] 移动端 App：地图显示、卫星消息收发
- [ ] 卫星通信协议对接（北斗 + 铱星）
- **验收**：设备可预测风险并告警，App 可查看位置与消息

---

### Sprint 5：SwarmOS 调度引擎
**目标**：AI 调度，支持 100+ 机器人
- [ ] 强化学习训练环境（Python + Gym）
- [ ] 训练 PPO 策略网络
- [ ] 实现 SwarmScheduler AI 版本（可切换规则/AI）
- [ ] 性能优化：调度延迟 < 100ms
- [ ] 压力测试：100 机器人仿真
- **验收**：在仿真中吞吐量提升 25%，相比规则调度

---

### Sprint 6：产品化与 OTA
**目标**：可交付版本，稳定可靠
- [ ] OTA 升级服务（差分更新，断点续传）
- [ ] 完善监控与告警
- [ ] 用户文档、API 文档
- [ ] 安全审计（渗透测试）
- [ ] 灰度发布流程
- [ ] 首批客户 POC 支持
- **验收**：系统可用性 99.9%，完成 5 家 POC 交付

---

### Sprint 7-12：后续路线
- **Sprint 7-8**：SwarmBot 产品集成（适配 3 家 AMR）
- **Sprint 9-10**：数据分析平台（报表、BI）
- **Sprint 11-12**：企业版功能（多租户、定制化）

## 🔄 敏捷仪式

### 每日站会（15 分钟）
- 昨天做了什么？
- 今天计划做什么？
- 遇到什么障碍？
- **形式**：异步更新（Slack/钉钉）+ 每日 10 分钟视频快速同步

### 冲刺规划会（1-2 小时）
- PO 讲解优先级最高的用户故事
- 团队估算（Planning Poker）
- 确定冲刺目标与 Sprint Backlog

### 冲刺评审（1 小时）
- 演示已完成功能
- 收集反馈，更新产品待办列表

### 冲刺回顾（1 小时）
- 哪些做得好？
- 哪些可以改进？
- 制定 1-2 项改进措施，下冲刺验证

## 📈 跟踪指标

- **速率（Velocity）**：每冲刺完成故事点，长期跟踪预测
- **累积流图**：在制品（WIP）分布，瓶颈识别
- **缺陷密度**：测试后发现的缺陷数，质量监控
- **部署频率**：生产环境发布次数，DevOps 成熟度
