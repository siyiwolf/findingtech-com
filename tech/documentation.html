部署方案

基于GitHub Actions + Docker + Kubernetes的CI/CD流水线。

开发 → 生产流程

  1. 代码提交 → 触发GitHub Actions
  2. 自动化测试 → 单元测试 + 集成测试
  3. 构建镜像 → Docker multi-stage build
  4. 安全扫描 → Trivy + Hadolint
  5. 部署到Staging → 自动集成测试
  6. 人工审批 → 生产环境部署

开发计划

敏捷开发框架下的冲刺计划和里程碑规划。

团队组织

  • 硬件团队（5人）：传感器、电路、结构设计
  • 固件团队（4人）：RTOS、驱动、AI推理
  • 后端团队（6人）：云平台、API、数据处理
  • 前端团队（3人）：Web控制台、移动App
  • 算法团队（4人）：导航、编队、决策
  • 测试团队（3人）：QA、自动化测试
