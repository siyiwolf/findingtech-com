API 文档

Finding Cloud 提供 RESTful API，用于与设备交互和数据获取。API 使用 HTTPS 和 Bearer Token 认证。

快速开始

  1. 在 开发者portal 注册并获取 API Key
  2. 使用下面的 cURL 示例测试连接
  3. 阅读完整文档，开始构建
# 获取设备列表 curl -H "Authorization: Bearer YOUR_API_KEY" \ "https://api.findingtech.com/v1/devices" # 发送短报文 curl -X POST "https://api.findingtech.com/v1/messages" \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{"device_id": "DEVICE123", "message": "OK"}' # 订阅 SOS 事件 (Webhook) curl -X POST "https://api.findingtech.com/v1/webhooks" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{"url": "https://your-server.com/sos", "event": "sos"}'
认证方式 设备管理 消息发送 位置查询 Webhook

开发者常见问题

Q: API 有调用频率限制吗？

A: 默认每秒 10 次请求。如有更高需求，联系我们申请提升。Webhook 没有频率限制。

Q: 数据存储多久？

A: 位置历史默认保留 365 天，传感器数据保留 90 天。企业客户可配置更长保留期。

Q: 可以在自己的服务器部署吗？

A: 我们提供私有化部署方案，适合大型组织和科研机构。请联系 sales@findingtech.com 咨询。

Q: SDK 是否支持实时位置流？

A: 支持。通过 WebSocket 或 Server-Sent Events (SSE) 可以接收设备实时位置推送，延迟低于 2 秒。

Q: 能否自定义 AI 模型？

A: 设备端支持 TensorFlow Lite 和 ONNX 模型。如果你有特定场景的需求（如特定动物识别），可以训练模型并加载到设备。联系我们获取工具链。
